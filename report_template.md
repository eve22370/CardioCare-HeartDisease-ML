# CardioCare 최종 보고서 초안

## 1. 문제 정의와 사용 목적
CardioCare는 임상 데이터로부터 심장병 발병 가능성을 예측하여 심장 전문의의 의사결정을 보조하는 시스템이다. 본 시스템은 환자의 진단을 단독으로 결정하지 않으며, 전문의에게 위험도 정보를 제공하는 보조 도구로 사용된다. 즉, 핵심 원칙은 “inform, not decide”이다.

## 2. EDA 핵심 결과
- `head()`, `info()`, `describe()`를 통해 데이터 구조와 변수 타입을 확인하였다.
- `target` 분포를 확인하여 클래스 불균형 가능성을 검토하였다.
- `age`, `trestbps`, `chol`, `thalach`, `oldpeak`에 대해 boxplot과 IQR 기준으로 이상치를 탐지하였다.

## 3. 전처리 결정
수치형 결측치는 중앙값으로 대치하고 StandardScaler로 표준화하였다. 범주형 변수는 최빈값으로 대치하고 OneHotEncoder를 적용하였다. 모든 전처리 변환은 학습 데이터에만 fit하여 데이터 누수를 방지하였다.

## 4. 모델 비교와 최종 선택
Logistic Regression, SVC, Random Forest, KNN을 비교하였다. 평가 지표는 balanced accuracy, precision, recall, F1, confusion matrix를 사용하였다. 의료 맥락에서는 False Negative가 특히 위험하므로 recall을 우선 고려하고 balanced accuracy로 보완하였다.

## 5. 테스트와 패키징
unittest로 예측 shape, 확률 범위, 임상 입력 범위, 결정론성을 검증하였다. Dockerfile을 통해 requirements.txt 기반 환경을 재현하고, 컨테이너 안에서 학습과 추론이 실행되도록 구성하였다.

## 6. 모니터링과 드리프트
추론 시 모델 버전, 입력 shape, 예측값, 실제값을 로그 파일로 남겼다. `chol`, `trestbps`, `oldpeak` 분포를 인위적으로 이동시킨 뒤 KS 검정으로 드리프트를 탐지하였다. p-value < 0.05인 특성은 드리프트 후보로 플래그하였다.

## 7. 재학습 및 피드백 루프
드리프트가 감지되고 balanced accuracy가 기준치 이상 하락하면 최신 데이터를 포함해 재학습을 수행한다. 단, 자동 재학습이 잘못된 예측을 다시 학습하는 폭주형 피드백 루프를 만들 수 있으므로, 재배포 전에는 전문의 또는 ML 엔지니어의 승인을 거친다.

## 8. 서빙 선택
Model-as-a-Service 방식을 선택한다. 병원 서버에서 모델을 중앙 관리하면 업데이트가 쉽고, PHI를 통제된 환경에서 관리할 수 있으며, 모바일 기기보다 보안과 감사 추적 측면에서 유리하다.

## 9. 한계와 윤리적 고려
이 프로젝트는 공개 데이터셋 기반이므로 실제 병원 환경의 환자군을 완전히 대표하지 못할 수 있다. 또한 모델은 진단자가 아니라 의사결정 보조 도구이다. 1주가 더 주어진다면 SHAP 기반 설명 가능성 분석과 외부 검증 데이터셋 평가를 추가할 것이다.
